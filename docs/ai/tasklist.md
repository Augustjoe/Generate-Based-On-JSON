# 项目任务路线图 (Tasklist Roadmap)

此文件由 Planner (GPT-5.5) 维护。Executor (Codex) 只能阅读此文件以了解其当前任务。

## 项目路线图 (Project Roadmap)

### 阶段 0: 项目理解 (Phase 0: Project Understanding)
- [ ] **Task 000**
  - **Status:** Pending (待办)
  - **Goal:** 分析现有代码库并建立基线上下文。
  - **Scope:** 只读分析。
  - **Files likely affected:** 无（仅限文档）
  - **Acceptance criteria:** 填充 `current_state.md`。
  - **Constraints:** 不要修改代码。
  - **Notes:** 初始设置阶段。

### 阶段 1: 项目设置 (Phase 1: Project Setup)
*(用于初始化脚手架、CI/CD、数据库 schema 的任务)*

### 阶段 2: 核心功能 (Phase 2: Core Features)
*(主要应用程序功能的任务)*

### 阶段 3: 测试和质量 (Phase 3: Testing and Quality)
*(致力于提高测试覆盖率和修复技术债务的任务)*

### 阶段 4: 打磨和发布 (Phase 4: Polish and Release)
*(UI 打磨、性能调优、部署)*

## 修复任务 (Bugfix Tasks)
*(保留给在 NEEDS_FIX 审查期间生成的任务)*

## 审查后续 (Review Follow-ups)
*(Reviewer 要求的不足以作为一个完整任务的微小调整)*

## 待办列表 (Backlog)
*(已计划但尚未安排到特定阶段的任务)*

## 未来想法 (Future Ideas)
*(未计划的功能、锦上添花的功能)*

## 已完成的任务 (Completed Tasks)
*(在任务获得 PASS 审查后移至此处)*

---

### 任务格式参考（供 Planner 使用）
```markdown
- [ ] **Task ID:** [例如，Task 001]
  - **Status:** [Pending (待办) / In Progress (进行中) / Blocked (已阻塞)]
  - **Goal:** [清晰的单句目标]
  - **Scope:** [对应该修改的内容的严格限制]
  - **Files likely affected:** [列出文件或目录]
  - **Acceptance criteria:** [所需结果的要点清单]
  - **Constraints:** [例如，"不要使用外部库"，"保持向后兼容性"]
  - **Notes:** [为 Executor 提供的任何有用的上下文]
```
